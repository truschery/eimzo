# Как внести свой вклад

Привет! Мы рады, что вы хотите помочь развитию проекта. Чтобы совместная работа была продуктивной и комфортной, пожалуйста, ознакомьтесь с этим коротким гайдом.

## 1. С чего начать?

Прежде чем писать код:
* **Проверьте Issues:** Возможно, баг, который вы нашли, уже обсуждается, или фича, которую вы хотите добавить, уже в работе.
* **Обсудите изменения:** Если вы хотите внести серьезные изменения или добавить новую функциональность, лучше сначала создать Issue и обсудить это. Это сэкономит ваше время, если идея не вписывается в планы проекта.

## 2. Процесс разработки

Мы используем стандартный **GitHub Flow**.

1.  **Fork & Clone:** Сделайте форк репозитория или клонируйте его
2.  **Branch:** Создайте новую ветку для вашей задачи от ветки `dev`.
    * Называйте ветки осмысленно.
    * Пример для новой фичи: `feature/add-something`
    * Пример для исправления бага: `fix/something`
3.  **Code:** Внесите изменения. Старайтесь делать коммиты небольшими и логически завершенными.
4.  **Commit:** Пишите понятные сообщения к коммитам на английском языке.
    * *Плохо:* "fix"
    * *Хорошо:* "Fix something validation error"
5.  **Push & Pull Request:** Запушьте ветку и создайте Pull Request (PR) в ветку `dev`.

## 3. Стиль кода

Чтобы код оставался читаемым:
* Придерживайтесь стиля, который уже используется в проекте.
* Не оставляйте закомментированный код и лишние пробелы.

## 4. Правила Pull Request (PR)

* Один PR должен решать одну задачу. Не смешивайте исправление багов и рефакторинг в одну кучу.
* В описании PR кратко расскажите, что было сделано и зачем.
* Если PR закрывает какой-то Issue, укажите это (например, `Closes #12`).
* Будьте готовы к Code Review. Правки — это нормальная часть процесса.

Спасибо за ваш вклад!
